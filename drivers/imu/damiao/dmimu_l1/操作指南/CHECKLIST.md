# DM-IMU L1 快速检查清单

## 📋 集成前检查

- [ ] PX4 源码已准备好
- [ ] 知道自己的飞控板型（如 fmu-v5, fmu-v6x）
- [ ] 有编译环境（已安装 PX4 工具链）
- [ ] IMU 硬件已到手

---

## 🔧 编译检查

- [ ] 驱动文件已复制到 `src/drivers/imu/dm_imu_l1/`
- [ ] 板级配置已添加 `CONFIG_DRIVERS_IMU_DM_IMU_L1=y`
- [ ] 编译成功，无错误
- [ ] 固件文件已生成（`.px4` 文件）

---

## 🔌 硬件连接检查

- [ ] CAN_H 连接正确
- [ ] CAN_L 连接正确
- [ ] GND 连接正确
- [ ] 电源连接正确（5V）
- [ ] 终端电阻已安装（120Ω）
- [ ] IMU 指示灯亮起（绿灯呼吸 = 正常）

---

## 🚀 首次启动检查

在 MAVLink 控制台执行：

```bash
# 1. 检查 CAN 设备
ls -l /dev/can*
```
- [ ] 看到 `/dev/can0` 或 `/dev/can1`

```bash
# 2. 启动驱动
dm_imu_l1 start
```
- [ ] 看到 "DM-IMU initialized" 消息
- [ ] 看到 "DM-IMU started in active mode" 消息
- [ ] 没有错误信息

```bash
# 3. 检查状态
dm_imu_l1 status
```
- [ ] State: 2 (主动模式)
- [ ] Error count: 0
- [ ] Active: yes

---

## 📊 数据验证检查

```bash
# 4. 查看加速度计
listener sensor_accel
```
- [ ] 数据持续更新（timestamp 增加）
- [ ] x, y, z 有合理的值
- [ ] z 轴约为 9.8 m/s²（静止水平时）
- [ ] error_count 为 0

```bash
# 5. 查看陀螺仪
listener sensor_gyro
```
- [ ] 数据持续更新
- [ ] 静止时 x, y, z 接近 0
- [ ] 转动 IMU 时数值变化
- [ ] error_count 为 0

```bash
# 6. 查看姿态
listener vehicle_attitude
```
- [ ] 四元数数据更新
- [ ] 水平静止时 q[0] 接近 1.0

---

## 🔧 校准检查

```bash
# 7. 校准加速度计（保持水平静止）
dm_imu_l1 accel_cal
```
- [ ] 命令执行成功
- [ ] IMU 指示灯闪烁（黄灯）
- [ ] 3-5 秒后恢复正常（绿灯）

```bash
# 8. 校准陀螺仪（保持完全静止）
dm_imu_l1 gyro_cal
```
- [ ] 命令执行成功
- [ ] IMU 指示灯闪烁
- [ ] 3-5 秒后恢复正常

```bash
# 9. 保存参数
dm_imu_l1 save
```
- [ ] 看到 "Saving parameters" 消息

---

## 🔄 重启测试

```bash
# 10. 重启飞控
reboot
```

等待重启完成后：

```bash
# 11. 重新启动驱动
dm_imu_l1 start
```
- [ ] 驱动正常启动
- [ ] 数据正常输出
- [ ] 校准参数已保存（数据稳定）

## 🐛 故障排查清单

如果遇到问题，按顺序检查：

### 驱动无法启动
- [ ] 检查 `/dev/can0` 是否存在
- [ ] 检查 CAN 驱动是否加载
- [ ] 尝试 `can start` 命令
- [ ] 检查编译配置是否正确

### 没有数据
- [ ] 确认驱动已启动（`dm_imu_l1 status`）
- [ ] 确认在主动模式（`dm_imu_l1 active_mode`）
- [ ] 检查硬件连接
- [ ] 检查 CAN ID 是否正确
- [ ] 检查 IMU 指示灯状态

### 数据超时
- [ ] 检查 CAN 总线波特率
- [ ] 检查终端电阻
- [ ] 检查线缆质量
- [ ] 重启驱动

### 数据异常
- [ ] 重新校准传感器
- [ ] 检查安装方向
- [ ] 恢复出厂设置
- [ ] 重启 IMU
