# FPV Quadtailsitter 模型使用说明

## 已完成的配置

### 1. 模型文件位置
- **Gazebo模型**: `PX4_Swarm/Tools/simulation/gz/models/fpv_quadtailsitter/`
  - `model.config` - 模型配置文件
  - `model.sdf` - 模型SDF描述文件
  - `meshes/` - 包含你的STL文件和螺旋桨模型

### 2. PX4配置文件
- **Airframe配置**: `PX4_Swarm/ROMFS/px4fmu_common/init.d-posix/airframes/4022_gz_fpv_quadtailsitter`
- **CMakeLists.txt**: 已添加到编译列表

## 启动指令

### 方法1：使用make命令（推荐）
```bash
cd PX4_Swarm
make px4_sitl_default gz_fpv_quadtailsitter
```

### 方法2：使用环境变量
```bash
cd PX4_Swarm
PX4_SIM_MODEL=fpv_quadtailsitter make px4_sitl_default gz
```

### 方法3：指定airframe ID
```bash
cd PX4_Swarm
make px4_sitl_default
# 然后在PX4 shell中:
# param set SYS_AUTOSTART 4022
```

## 模型参数说明

该模型使用了与 `quadtailsitter` 完全相同的飞行参数：

### 飞行控制参数
- **机型类型**: VTOL Quad Tailsitter (立式垂直起降四旋翼)
- **质量**: 1.6 kg
- **旋翼配置**: 4个旋翼（前后各2个）
- **飞行模式**: 支持多旋翼模式和固定翼模式切换

### 关键参数
- `VT_TYPE 0` - 尾座式VTOL
- `MAV_TYPE 20` - VTOL类型
- `CA_AIRFRAME 4` - 四旋翼配置

## 自定义修改

### 如果需要调整模型外观
编辑文件: `PX4_Swarm/Tools/simulation/gz/models/fpv_quadtailsitter/model.sdf`

在 `<visual name='base_link_visual'>` 部分修改：
```xml
<geometry>
  <mesh>
    <scale>0.001 0.001 0.001</scale>  <!-- 调整缩放比例 -->
    <uri>model://fpv_quadtailsitter/meshes/你的STL文件名.STL</uri>
  </mesh>
</geometry>
```

### 如果需要调整旋翼位置
在 `model.sdf` 中修改各个 `rotor_X` 的 `<pose>` 标签：
```xml
<link name='rotor_0'>
  <pose>0.145 -0.23 0.46 0.17 -0.17 0</pose>  <!-- X Y Z Roll Pitch Yaw -->
  ...
</link>
```

### 如果需要调整飞行参数
编辑文件: `PX4_Swarm/ROMFS/px4fmu_common/init.d-posix/airframes/4022_gz_fpv_quadtailsitter`

常用参数：
- `FW_THR_TRIM` - 固定翼油门配平
- `MC_ROLL_P` - 多旋翼横滚P增益
- `VT_ARSP_TRANS` - 转换空速

## 重新编译

修改任何配置后，需要重新编译：
```bash
cd PX4_Swarm
make clean
make px4_sitl_default gz_fpv_quadtailsitter
```

## 故障排查

### 如果模型不显示
1. 检查STL文件路径是否正确
2. 尝试调整 `<scale>` 参数（可能模型太大或太小）
3. 检查Gazebo日志: `~/.gz/sim/server.log`

### 如果飞行不稳定
1. 检查旋翼位置是否正确
2. 调整质量和惯性参数
3. 参考原始 `quadtailsitter` 模型进行对比

## 当前使用的STL文件
主机身: `Assem1 - FPVZW.stp-1 FPV.1.1.stp-1.STL`

如需更换其他STL文件，修改 `model.sdf` 中的 `<uri>` 路径即可。
