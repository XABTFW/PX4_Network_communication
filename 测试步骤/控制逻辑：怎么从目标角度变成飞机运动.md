1. 控制逻辑：怎么从目标角度变成飞机运动
    这套方案的输入不是“目标坐标”，而是“目标相对导引头看过去偏了多少角度”。文档里对应的是 DYT 回传的脱靶角、云台方位/俯仰角、角速度和锁定状态，PX4%20+%20DYT%20导引头视
    觉拦截伴飞技术方案.md:15。

  第一步是重建 LOS，也就是目标的视线方向。做法是先在导引头坐标系里根据脱靶角构造一个单位向量，再乘上“导引头到机体”的姿态变换，再乘上“机体到 NED”的姿态变换，得到世界
  系里的 u^n。PX4%20+%20DYT%20导引头视觉拦截伴飞技术方案.md:68 这里有个关键点：导引头是两轴稳像，所以这个变换矩阵不是常数，必须实时带入云台当前的俯仰和方位角。

  第二步是求 LOS 角速度。PN.md 里给的是最小实现：先对 LOS 做低通，再做差分得到 du/dt，然后算 omega_LOS = u x du/dt。PN.md:43 这个量可以理解成“目标视线在你眼前扫动得
  有多快、往哪边扫”。

  第三步是构造近似比例导引。核心公式在 PN.md:128：

  a_cmd = N*Vhat_c*(omega_LOS x u) + k_a*u + k_v*(v_cmd*u - v_m)

  这 3 项作用不同：

  - N*Vhat_c*(omega_LOS x u)：PN 主项，负责横向修正，让飞机别傻追尾，而是往更合理的闭合方向机动。
  - k_a*u：沿视线推进项，保证飞机确实朝目标方向压过去。
  - k_v*(v_cmd*u - v_m)：速度阻尼项，让速度朝期望方向收敛，避免发散和振荡。

  因为没有距离，所以真实闭合速度算不出来，文档用“本机速度在视线方向上的投影”替代，叫伪闭合速度 Vhat_c。PN.md:65 这不是严格物理量，但足够让控制律先工作起来。

  最后不是直接把 a_cmd 硬塞给最底层姿态环，而是映射成 trajectory_setpoint，以“速度设定 + 加速度前馈 + yaw”为主。PX4%20+%20DYT%20导引头视觉拦截伴飞技术方案.md:40 这
  意味着：

  - PX4 现有位置控制器继续负责姿态、推力、限幅和稳定性。
  - 视觉模块只负责“告诉飞机往哪飞、飞多快、头朝哪”。

  2. PX4 里怎么落地
    文档推荐的不是外挂模块，而是正式接入 PX4 FlightTask 链路，一、总体实现思路.md:3。整体分 4 层：

  1. DYT 驱动层
     路径建议在 src/drivers/dyt_gimbal/，一、总体实现思路.md:79。职责很纯粹：UART 收发、协议同步、CRC、重连、发送“识别转跟踪”命令、解析表 1/表 2、发布 uORB。
     这一层不要做飞行几何计算，只做“把字节流变成结构化观测”。
  2. dyt_target uORB 层
     这里发布的是导引头原始/半结构化观测，比如 tracking_state、los_az_rad、los_el_rad、gimbal_yaw_rad、gimbal_pitch_rad、timestamp_sample 等，一、总体实现思
     路.md:125。
     这样做的好处是驱动层不依赖飞控姿态，不跟控制层强耦合。
  3. DytLosProcessor 观测处理层
     文档建议把它作为 FlightTask 内部 helper，而不是另起系统模块，一、总体实现思路.md:170。
     它负责：

  - 角度转 LOS 向量
  - 时间戳修正
  - 固定延迟补偿
  - 帧年龄和抖动检查
  - 低通、差分、短时预测
  - 输出 u_n / du_dt / omega_los / is_fresh

  4. FlightTaskAutoDytFollow 控制层
     这是最核心的模式任务，一、总体实现思路.md:212。它订阅目标观测、本机姿态、位置、速度和 RC 输入，然后维护状态机并生成 trajectory_setpoint。
     文档里的状态机是：

  - IDLE
  - SEARCH_WAIT_LOCK
  - TRACK_FOLLOW
  - TRACK_INTERCEPT
  - LOST_HOLD
  - ABORT
    见 PX4%20+%20DYT%20导引头视觉拦截伴飞技术方案.md:33

  对应运行流程可以理解成：

  1. RC 打开视觉模式。
  2. 发一次 DYT 跟踪触发命令。
  3. 等待连续锁定若干帧。
  4. 先进入 FOLLOW。
  5. 条件够好时允许切到 INTERCEPT。
  6. 一旦锁丢了或者时延异常，退回 LOST_HOLD。
  7. 人工接管或 failsafe 时直接 ABORT。

  3. 实飞调参与验证顺序
    这部分最重要，因为这类方案通常不是公式先出问题，而是方向、时延、限幅和状态机先出问题。

  第一阶段先不追目标，只做几何和通信验证：

  - 串口收发、帧同步、CRC、超时恢复是否正常。PX4%20+%20DYT%20导引头视觉拦截伴飞技术方案.md:140
  - 云台方位/俯仰正方向是否和 PX4 机体系定义一致。PX4%20+%20DYT%20导引头视觉拦截伴飞技术方案.md:127
  - 导引头零位是否真的是机头前向。
  - 静止目标时，目标在左边，算出来的 LOS 是否也指向左边；目标抬高，LOS 的仰角是否同步变大。

  第二阶段测时延，不要急着上 INTERCEPT：

  - 文档明确要求实测 DYT 总延迟和抖动。PX4%20+%20DYT%20导引头视觉拦截伴飞技术方案.md:64
  - 方法一般是快速摆动目标或摆动转台，看图像事件和飞控接收时间差。

  第三阶段只调 FOLLOW：

  - k_v_follow 先大一点，保证阻尼。
  - k_a_follow 只给足够的前推，不要太猛。
    目标是先验证“稳稳朝着目标逼近，不蛇形、不抽动、不丢锁后乱冲”。

  第四阶段再开 INTERCEPT：

  - 提高 N_intc
  - 提高 v_cmd_intc
  - 适当加大 k_a_intc
  - 但要同时收紧最大倾角、最大水平速度、最大下降率。PX4%20+%20DYT%20导引头视觉拦截伴飞技术方案.md:84

  第五阶段重点看日志，而不是只看肉眼效果：

  - tracking_state
  - 原始 LOS 和补偿后 LOS
  - frame_dt、rx_age
  - omega_los
  - a_cmd
  - trajectory_setpoint
  - 饱和和退出原因
    这些在文档测试计划里已经点出来了，PX4%20+%20DYT%20导引头视觉拦截伴飞技术方案.md:156。

  如果把这套方案再压缩成一句工程判断，就是：

  它本质上是一套“基于目标角度的视觉追踪飞行任务”，FOLLOW 是拿来把链路飞稳，INTERCEPT 是在链路稳定、时延可控之后再逐步放开的激进子模式。

  3. 控制律图

  输入量
  │
  ├─ 来自 DYT:
  │   - los_az, los_el
  │   - gimbal_yaw, gimbal_pitch
  │   - tracking_state
  │
  ├─ 来自 PX4:
  │   - 机体姿态 C_b^n
  │   - 本机速度 v_m^n
  │   - 本机位置/高度
  │
  └─ 来自参数:
      - 安装外参 C_g^b
      - delay
      - N / k_a / k_v / v_cmd
      - 限速/限加速度/限倾角

  步骤 1: 重建 LOS 方向
  导引头脱靶角
     ▼
  导引头坐标系 LOS 向量 u_g
     ▼ 乘实时云台姿态 + 安装外参
  机体系 LOS 向量 u_b
     ▼ 乘飞机姿态
  NED 系 LOS 向量 u_n

  步骤 2: 时间对齐与延迟补偿
  接收时刻 t_rx
     ▼
  测量时刻 t_meas = t_rx - DYT_PIPELINE_DELAY
     ▼
  按观测时间排序缓存
     ▼
  低通后的 u_f
     ▼
  du/dt ≈ (u_f(k) - u_f(k-1)) / dt_meas
     ▼
  当前时刻预测:
  u_now ≈ u_meas + du/dt * dt_delay

  步骤 3: 计算 LOS 角速度
  omega_LOS = u x du/dt

  步骤 4: 构造伪闭合速度
  Vhat_c = max(V_min, v_m^n · u)

  步骤 5: 计算导引加速度
  a_cmd =
      N * Vhat_c * (omega_LOS x u)
    + k_a * u
    + k_v * (v_cmd * u - v_m)

  步骤 6: 映射为 PX4 setpoint
  u 的水平投影  ─────► yaw_sp = atan2(u_y, u_x)
  u 的方向      ─────► velocity_sp
  a_cmd         ─────► acceleration_ff
  高度策略      ─────► z_sp 或 vz_sp
  保护约束      ─────► 限速/限加速度/限倾角/限下降率

  FOLLOW
  - 小 N
  - 小 v_cmd
  - 较大 k_v
  - z 方向保守
  - yaw 变化保守
  - 目标: 稳定跟随、稳定逼近、不振荡

  INTERCEPT
  - 大 N
  - 大 v_cmd
  - 较大 k_a

  - 但要求:
    - 连续稳定锁定
    - 视觉时延在阈值内

    - 目标位于前向锥内
    - 倾角/速度/高度余量足够
  - 目标: 快速闭合，但仍受保护约束

  6. 团队讨论时最值得盯住的 6 个问题

  - DYT 回传角度的正方向、零位、轴定义是不是和 PX4 机体系完全一致。
  - 延迟补偿是按“观测时间”算，还是按“接收时间”直接算。
  - INTERCEPT 的准入条件是否足够严格。
  - 丢失目标时到底是“悬停等待”还是“退回位置模式”。
  - 高度策略第一版是否只做保守跟随，不做激进下冲。
  - 控制输出是否只通过 trajectory_setpoint 进入 PX4，不旁路底层控制。
